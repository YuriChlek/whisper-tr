Як очищати память докер перед новим білдом?
Для очищення пам'яті Docker перед новим білдом існує кілька корисних команд. Ось основні способи, які ви можете використовувати:
Очищення невикористаних об'єктів:

docker system prune
Ця команда видалить всі зупинені контейнери, невикористані мережі, висячі образи та невикористаний кеш збірки.
Prune unused Docker objects надає детальну інформацію про цю команду.
Якщо ви також хочете видалити невикористані томи, додайте прапорець --volumes:

docker system prune --volumes
Для видалення всіх невикористаних образів (не тільки висячих), використовуйте:

docker image prune -a
Prune images містить більше інформації про цю команду.
Якщо ви використовуєте Docker Compose, ви можете видалити всі ресурси, пов'язані з вашим проектом:

docker-compose down --rmi all --volumes
Це зупинить і видалить контейнери, мережі, томи та образи, створені docker-compose up.
Для очищення кешу збірки Docker BuildX (якщо ви його використовуєте):

docker buildx prune
Managing build cache містить додаткову інформацію про управління кешем збірки.
Важливо зазначити, що ці команди можуть видалити дані, які вам можуть знадобитися пізніше. Тому перед їх використанням переконайтеся, що ви розумієте, які саме ресурси будуть видалені.
